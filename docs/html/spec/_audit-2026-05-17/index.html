docs/ 監査レポート 2026-05-17

メタ ISSUE: #1124 — Harmony 仕様書・プレゼン HTML 化プロジェクト Phase A: #1125 — docs/ 監査・クリーンアップ

監査の目的

Harmony の仕様書群 (docs/spec 61 + docs/setup 3 + docs/user-guide 6 + docs/conventions 3 = 計 73 files / 23,402 行) を HTML 化するメタプロジェクト #1124 の前提作業。HTML 化後の二重メンテ負担を最小化するため、現状不一致 / 陳腐化 / 重複 / 不要な md を洗い出してクリーンアップする。

方法

Sonnet サブエージェント 3 並列で領域別 audit を実施。

  • Agent 1: docs/spec/ アルファベット順前半 (A-G、30 files)
  • Agent 2: docs/spec/ アルファベット順後半 (M-W、31 files)
  • Agent 3: docs/setup + docs/user-guide + docs/conventions (12 files)

各 agent は AGENTS.md / CLAUDE.md / 主要 spec を読込んで判断軸を確立した上で、5 分類 (PASS / 陳腐化 / 重複 / 不要 / 判断保留) を実施。削除候補は grep -rn で他参照ゼロ確認済み。

判断は Opus 親が統合し、削除 / 修正は Opus 監督下で Sonnet が実施。
