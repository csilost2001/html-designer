## 監査の目的

Harmony の仕様書群 (docs/spec 61 + docs/setup 3 + docs/user-guide 6 + docs/conventions 3 = 計 73 files / 23,402 行) を HTML 化するメタプロジェクト #1124 の前提作業。HTML 化後の二重メンテ負担を最小化するため、現状不一致 / 陳腐化 / 重複 / 不要な md を洗い出してクリーンアップする。

## 方法

Sonnet サブエージェント 3 並列で領域別 audit を実施。

- Agent 1: docs/spec/ アルファベット順前半 (A-G、30 files)
- Agent 2: docs/spec/ アルファベット順後半 (M-W、31 files)
- Agent 3: docs/setup + docs/user-guide + docs/conventions (12 files)

各 agent は AGENTS.md / CLAUDE.md / 主要 spec を読込んで判断軸を確立した上で、5 分類 (PASS / 陳腐化 / 重複 / 不要 / 判断保留) を実施。**削除候補は `grep -rn` で他参照ゼロ確認済み**。

判断は Opus 親が統合し、削除 / 修正は Opus 監督下で Sonnet が実施。

## 5. Phase B 技術スタック確定 (audit と並行調査済み)

別 agent (Phase B Tech Research) が並行調査済み、`agent-4-phase-b-research.md` (内部用) に詳細記録。要点:

| 項目 | 採用 | バージョン | 備考 |
|---|---|---|---|
| Astro | `astro` | ^6.3.3 | Node 22+ 必須 |
| Tailwind | `@tailwindcss/vite` | ^4.x | `@astrojs/tailwind` は deprecated |
| Tailwind typography | `@tailwindcss/typography` | ^0.5.x | CSS `@plugin` 構文で v4 対応 |
| dark mode | CSS `@custom-variant` | — | `tailwind.config.js` 不要 |
| pagefind | `astro-pagefind` | ^2.0.0 | post-build 自動インデックス、native web components UI |
| mermaid | `rehype-mermaid` | ^3.0.0 | Playwright 依存あり / `strategy: 'inline-svg'` で静的 SVG |
| shiki | Astro 同梱 (Shiki 4) | — | json/ts/bash/dockerfile/mermaid バンドル済 |
| copy button | 自前 `<script>` | — | stable npm package なし |
| SchemaTable | 自前 Astro component | — | 再帰的 properties 展開の skeleton 提示済み |

注意:
- `rehype-mermaid` v3 は Playwright 必須。Dev Container では既存インストール済みだが build 前に確認推奨
- `outDir: '../docs/html'` は相対パス指定可能、pagefind index も自動的にその配下に生成
- Astro 6 は Node 22+ 必須のため、Dockerfile の Node バージョン要確認
